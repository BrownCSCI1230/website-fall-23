Project 1: Brush

  • Stencil Code
    • Fix clearCanvas()... why does it make the canvas bigger lmao
    • Make project 1 and lab 2 work the same way re: mouseDragged() requiring a left mouse button (LMB) check. They'll have to implement the LMB-is-down boolean themselves.
  • General
    • Specify that students should paint on mouseDown(). This applies to lab 2 as well, and both handouts should be edited.
      • This has an added bonus: we can grade their mask just by clicking once.
    • Specify that if your brush center is off-canvas, but its radius is big enough that it should be drawing on the canvas, then it must draw on the canvas.
  • Efficiency
    • Emphasize that they should only re-create the brush mask when necessary, i.e. they must check for changes in radius and brush type.
      • Modify gradescope rubric's efficiency part to account for this.
  • Constant
    • Fix constant brush.
      • EITHER:
        • Specify that constant brush should draw 1 pixel at radius 1, i.e. distance < radius;
        • Specify that constant brush should draw 5 pixels at radius 1, i.e. distance ≤ radius; or
        • (Preferred) use this nifty formula: distance < radius - sqrt(2)/2. It produces a much nicer circle, that looks like the kind a pixel art app would produce. Ask zcheng12 or dtoth1 for details.
      • Important: this will require changing all examples of the constant brush (videos, images, etc)!
    • Possibly move constant brush into lab 2: pencil, as an optional final task. This would mean that students can collaborate on it, so weigh the tradeoffs accordingly. As it is, brush is a little too easy.
  • Smudge
    • Fix smudging from edges: use transparent (RGBA{0,0,0,0}) instead of black.
    • In the handout, warn them not to re-create their data structure every time mouseDragged() is called. It's especially easy to forget that clear() will cause this to happen, requiring a reallocation of memory afterwards.
    • Specify to use linear or quadratic (please pick one) for smudging, to make our grading easier. Do not permit using the constant brush.
    • (Recommended, but optional) Change project requirements so that smudge brush must be affected by alpha.
      • In this case, we can say: when alpha is 255, however much I smudge in a single stroke (within the canvas), the pixel under the cursor should always stay the same.
  • Spray
    • Emphasize that spray should spray dots of the same color, i.e. THE BRUSH COLOR, accounting for the alpha.
    • Specify that spray should do a DIFFERENT distribution of points every time it is "painted" to the screen.
    • Specify that spray should fill density proportion of its mask: if density = 100, it's basically constant brush; if density = 0, it does nothing
    • Specify that the brush should be a circle centered on the mouse x,y
    • Make a note about using rand() or arc4_random()
  • Fill
    • Specify that it should fill on mouse down, not on mouse dragged.
    • Specify that fill should fill the canvas if clicked on an empty canvas; in general, it should treat the canvas bounds as... bounds.
  • Alpha Blending
    • Specify the correct implementation of alpha blending: the "effect" of the brush is "capped" at its current alpha value. This is what Photoshop does, though Procreate might behave differently.
    • (Optional) specify that it's fine for alpha blending not to work with their other extra credit brushes. Possibly give more extra credit if they fix those too.

Project 2: Filter

  • General
    • Be clearer in the handout about what we expect their filters to produce at edge cases, e.g. when blurring sigma = 0.
    • Change function names for Project 2: Filter in the projects_2D stencil, to match those in Lab 3: Convolution
  • Scaling
    • Specify in the handout to round the output size if the scale factor times the width or height produces a non-integer value. floor is fine, too, but be consistent.
      • It's very important to be consistent. Many students messed this up, causing a 45° shearing of their image due to an indexing mismatch of 1 pixel per row.
    • When at boundaries, instead of using getPixelReflected() or whatever, ask students to divide by the sum of the weights used when summing over the triangle. This is possibly more correct, but feel free to disagree.
  • Edge Detection
    • Fix sensitivity and be clearer about what it should do.
  • Extra Credit
    • Reduce point values for:
      • Basic chromatic aberration: 4 --> 3
      • Spatially-aware: 8 --> 6
  • Rotation
    • Specify in the handout that, as with all filters, rotation should be able to be applied consecutively on the same image!
